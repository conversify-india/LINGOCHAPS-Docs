3. Review Path Decision

Senior Decision Point

Determine the Review Path

After distribution, a Senior will assign the review methodology based on the project complexity and tool requirements.

Method A

Phrase TMS Path

Used for large or complex localization projects.

  • Work is done in the Phrase TMS tool
  • Includes automated QA checks, tag handling, and term consistency
  • Suitable for projects that require high accuracy and structured workflows
Tool-Driven Workflow

Method B

Direct Human Path

Used for standard or simpler projects.

  • Work is done manually on files (Word, Excel, PDF)
  • No need to use Phrase TMS
  • Easy to manage and can be done offline
  • Suitable for clients who don’t require tool-based workflows
Manual Quality Check
MANDATORY PROTOCOL

Standard Revision Protocol

Once the resource starts working on the file, perform an initial check to ensure the work is being done manually and not using Machine Translation (MT). If MT usage is detected, follow this process:

1

Contact Resource

Ask for clarification regarding the suspected MT usage immediately.

2

Inform Your Senior

Escalate the issue without delay to ensure management is aware.

3

Take Action

Follow instructions provided by your senior to resolve the situation.

Decision Guidance (Consult with Senior):

  • Ask the resource to revise and correct the completed portion.
  • If the resource is unable/unwilling to fix it:
    • Reassign the entire task, OR
    • Get completed part reviewed/corrected, and assign the remaining work to another resource.

4. Quality Control (QC) & Review

STEP 1 — CRITICAL

Google / AI Copy Check

Ensure translation is human-quality — not copied from any AI or translation tool.

What You Do

Take sample sentences from translated file
Cross-check using Google Translate
Cross-check using ChatGPT
Compare: structure, word choice, flow

MT Handling Protocol

If Machine Translation is detected during verification, apply the Standard Revision Protocol immediately:

1. STOP & FLAG 2. SEND BACK 3. ESCALATE 4. RE-CHECK

This unified protocol applies to both Phrase and Direct review paths.

Red Flags

Exact match with Google output
Robotic / unnatural phrasing
Same mistakes typical of machine translation

Unbreakable Rules

  • Never skip QC: This applies even for small 50-word tasks.
  • Zero MT Policy: Word-for-word Google/AI copies will be rejected.
  • Independent Review: Reviewer MUST be different from the translator.
  • Domain Match: Prefer expert reviewers for technical/legal files.

Decision Logic

Case 1: MT Detected

STOP → Inform Translator → Ask for Manual Revision
"The translation appears tool-generated. Please revise manually."

If Translator Agrees

Wait for revised file → Re-check for MT → Continue normal QC

If Translator Denies / Refuses

Escalate to Senior → Decide:
Option A: Full Re-translation
Option B: Review & Fix Only

Reassignment Rule

Even if work is reassigned — repeat the Google/AI check from scratch. No shortcuts. This is where most teams fail.

4. Self-Assessment & 5. Final Delivery

Step 4

Final Safety Check

"The last line of defense. Perform a side-by-side comparison to guarantee perfection."

🧠 Internal Checklist

  • Client instructions have been followed.
  • Formatting is 100% accurate and consistent with the source.
  • File type matches the required format.
  • No missing content or translation errors.
  • Naming convention is followed as per the details provided in the source document.

Pro Tip

Open both the source file and the final translated file side by side. Review different sections of the document to identify any signs of machine translation or AI usage, ensuring the translation has been done manually as required.

After that, go through the document page by page to check for any missed segments, formatting issues, or translation errors.

STEP 5

Professional Handover

Client Draft Review (Group B)

For Bank/Misc docs, share a draft with the client first to ensure no misunderstandings before the final handover.

The Dual-Format Rule

Deliver both .DOCX (editable) and .PDF (finalized). Ensure both match the reviewed version.

Naming Protocol

Format: ClientName_LanguagePair_Final.docx

The Ultimate Phrase TMS Guide

Complete onboarding for new reviewers and project managers.

What is Phrase TMS?

A cloud-based Translation Management System (formerly Memsource) where all Lingo Chaps localization happens.

Centralized

Access all Translation Memories (TM) and Term Bases (TB) in real-time from any browser.

Automated

Automated QA checks catch errors that human eyes might miss, ensuring technical perfection.

Collaborative

Multiple reviewers can work on the same project with live synchronization of edits.

1

Linguist Portal & Setup

  • Welcome Email: You will receive an invitation. Set your password and log in at cloud.memsource.com.
  • The Jobs View: Your dashboard shows all assigned tasks. Check the Status, Deadline, and Wordcount immediately.
  • Accepting Jobs: Change the status to "Accepted" before you begin working to notify the Project Manager.
2

Mastering the Web Editor

Interface Breakdown

Center Pane: The Translation Editor where you edit segment by segment.
Right Pane (CAT): Shows TM matches, Terms, and MT suggestions.
Bottom Pane: Real-time "In-Context Preview" of the document layout.
3

Quality Assurance (QA)

Before completing any job, you MUST run the automated QA check.

Common QA Warning Types:

  • ⚠️ Tags: Missing or mismatched formatting tags.
  • ⚠️ Terminology: Used a term not approved in the Term Base.
  • ⚠️ Inconsistency: Same source translated differently elsewhere.
4

Final Delivery

Confirm All Segments: Ensure every segment is ticked (confirmed).
Export Preview: Download the "Completed File" to verify final layout.
Update Status: Set the job to "Completed" in the portal.
