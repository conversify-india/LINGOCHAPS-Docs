Rule (Group A Only): Create a separate language folder for every language in the task. If a project has 3 languages, there should be 3 folders inside the date folder. Never mix languages in the same folder unless the documents are short and easy to manage. (Not for Misc clients).
Rule (Miscellaneous + Walk-in): If work from multiple clients (Miscellaneous + walk-in clients) is received on the same date in different languages, first create a language-wise folder (as shown above). Then, within each language folder, create separate folders for each client by their name. This ensures that checking and referencing files later is easy and organized.

3. Quality Control & Review Operations

A systematic, zero-tolerance framework designed to filter out linguistic inconsistencies and ensure every segment meets our premium delivery standards.

ACTION PROTOCOL

Standard Revision Protocol

MEANING

The content must be identical to the source while reading naturally in the target language. No literal translations that lose context.

TERMINOLOGY

Glossaries and client instructions are project-dependent and must be followed strictly. These requirements vary per task and are not uniform for all projects.

STYLE

Tone, flow, and natural phrasing must match the client expectations.

GRAMMAR

Spelling, syntax, and punctuation must be error-free. Ensure correct capitalization and localized formatting for dates/numbers.

STEP 1 — CRITICAL

Google / AI Copy Check

Live Review Protocol

Do not wait for project completion. Once real-time work progress of translator is avaialble, proceed with the Google Check immediately once work commences to detect issues at the source.

Ensure translation is human-quality — not copied from any AI or translation tool.

What You Do

Take sample sentences from translated file
Cross-check using Google Translate
Cross-check using online tools

MT Handling Protocol

1. STOP & FLAG 2. SEND BACK 3. ESCALATE

Unbreakable Rules

  • Never skip QC: Applies even for small 50-word tasks.
  • Zero MT Policy: Word-for-word copies will be rejected.
  • Independent Review: Reviewer ≠ Translator.
STEP 2

Translation Accuracy Check

"Verify excellence before the client sees it. Our internal QC is the final barrier against errors."

Reviewer Tip

For specialized files (Legal, Medical), always verify domain-specific terms using an authoritative glossary — never rely on intuition alone.

What to Verify

Meaning correct — no literal mismatch
No missing content / skipped segments
Numbers, names, dates accurate
Terminology fits project requirements

Reviewer Operations — Step-by-Step

MANDATORY WORKFLOW FOR PHRASE TMS PROJECTS

1
File Preparation

Clean the file before upload. Fix formatting issues and remove unnecessary line breaks (use tools if needed).

2
Project Setup in Phrase

Create the project in Phrase TMS and upload the prepared file to the workspace, attach TM, TB and then run it.

3
Task Assignment

Assign the file to the relevant translator/resource. Ensure they have access to relevant glossaries.

4
Work Tracking

Monitor progress in Phrase. Check for any machine translation usage (follow internal MT detection process). After this editing proofreading will be carried out by outsource.

5
Quality Check (QA)

Run QA checks in Phrase. Review errors (tags, numbers, terminology) and add comments where required & inform translator to make corrections accordingly.

6
File Extraction

Extract the final reviewed file. Naming Convention: Add the target language code to the file name (e.g., Project_Manual_DE.pdf). (If Pre processed by DTP then sent again for final review, if required)

7
Final Review

Perform a final manual review of the exported file to ensure 100% quality and accuracy.

Phrase TMS Guide

Master the core tool of our localization ecosystem.

The Linguistic Engine

Phrase TMS is our central processing hub. It synchronizes Source Files with our global Translation Memories (TMs) and Term Bases (TBs) in real-time. This ensures consistency across every project and maximizes cost-efficiency through word-count leverage.

The Quality Loop

Every segment undergoes an automated Quality Assurance (QA) check. Our protocol demands a 100% resolution of all technical warnings (Tags, Terminology, Formatting) before any file is approved for delivery. No exceptions.

Linguistic Asset Vault

This is where Lingo Chaps' "Institutional Memory" resides. By maintaining clean, high-quality TMs, we ensure that specialized terminology is preserved over years of collaboration, creating a competitive advantage for our clients.

Collaboration Hub

Phrase provides the workspace where Project Managers and Linguists interact directly. Using In-context Previews and Segment-level Comments, we resolve ambiguities instantly, ensuring the final output matches the client's intent.

Ecosystem Strategic Role

Think of Phrase as our Technical Ground Zero. While Zoho manages the business logic (tasks/finance) and WhatsApp/Email manage the people, Phrase is the only place where the Linguistic Truth of a project exists.
